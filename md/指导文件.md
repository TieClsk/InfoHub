
# InfoHub 个人信息网页 — 项目指导文件

## 一、项目概述

聚合六大板块信息的个人网页，采用"广泛采集 → AI 筛选整理"的模式：多数据源采集原始数据入库，由 DeepSeek 进行去重、排名、摘要、分类等智能处理，前端展示经 AI 整理后的高质量内容。

### 技术栈

- **前端**: Next.js 14+ (App Router) + Tailwind CSS + shadcn/ui
- **后端**: Next.js Route Handlers
- **数据库**: PostgreSQL + Prisma（双库：Main DB 存精选内容，Raw DB 存原始数据）
- **AI**: DeepSeek V4 Pro API
- **定时任务**: node-cron / Vercel Cron
- **部署**: Docker / Vercel

### 核心架构

```
多数据源 → Raw DB（原始数据，14天自动清理）
              ↓
     DeepSeek V4 Pro 处理
              ↓
     Main DB（精选内容，长期保留，冗余来源信息）
              ↓
           前端展示
```

## 二、数据源矩阵

### 国内热点

| 数据源 | 接入方式 | 内容特征 |
|--------|----------|----------|

| 今日热榜 tophub.today | 爬取 | 聚合知乎/微信/百度等多平台 |
| 澎湃新闻 RSS | rss-parser | 深度报道，时政为主 |

|--------|----------|----------|

| BBC News RSS | rss-parser | 英国视角，权威 |
| Reuters RSS | rss-parser | 财经+国际，权威 |
| Google News RSS | rss-parser | 覆盖面最广 |

### AI 领域新闻

| 数据源 | 接入方式 | 内容特征 |
|--------|----------|----------|
| 机器之心 RSS | rss-parser | 中文 AI 媒体头部 |

### GitHub 热榜

| 数据源 | 接入方式 | 内容特征 |
|--------|----------|----------|
| GitHub Trending 页面 | cheerio 爬取 | 官方热榜，按语言/时间维度 |
| GitHub Search API | REST API（可选 Token） | 按 Star 增量搜索，灵活度高 |

### 投资资讯

| 数据源 | 接入方式 | 内容特征 |
|--------|----------|----------|
| 东方财富 API | REST（非官方） | A 股行情 + 资讯 |
| 财联社快讯 | 爬取 | 实时财经快讯 |
| CoinGecko API | REST API（免费） | 加密货币行情 |
| 雪球热帖 | 爬取 | 投资观点+讨论 |

### 处理流程

```
所有数据源 → 写入 Raw DB（保留原始数据 + 源端排名/热度，14 天后自动清理）
                    ↓
         DeepSeek V4 Pro 批量处理
         - 去重：语义相似度判断是否同一事件
         - 排名：按重要性/热度评分 1-10（参考 sourceRank 作为权重依据）
         - 摘要：生成统一格式的中文摘要
         - 分类：细分标签（如 AI新闻→大模型/芯片/应用/论文）
         - 翻译：英文标题和摘要翻译为中文
                    ↓
         写入 Main DB ProcessedContent 表（含 sourceId + sourceName，独立于原始数据）
```

## 三、开发阶段

### Phase 0：初始化（0.5 天）

- 创建 Next.js 项目（TypeScript, App Router）
- 安装 Tailwind + shadcn/ui + Prisma
- 配置 ESLint、`.env.local`、Git

**验收：** `npm run dev` 启动正常，Prisma 连接数据库成功。

---

### Phase 1：数据库设计（0.5 天）

核心设计原则：**双库分离（Main DB 存精选 + Raw DB 存原始/14天清理），ProcessedContent 冗余来源信息确保独立可展示，JSON 扩展字段保障扩展性。**

详见 CLAUDE.md 中的数据库模型设计。

**验收：** 迁移成功，Prisma Studio 查看表结构正确，能手动插入测试数据。

---

### Phase 2：数据采集层（2-3 天）

**开发顺序：**
1. 先写通用 fetcher 基类/工具函数（超时、重试、日志）
2. GitHub Trending 爬取（最简单，验证全流程）
3. RSS 类 fetcher（机器之心、BBC 等，共用解析逻辑）
4. REST API 类 fetcher（天行数据、NewsAPI）
5. 爬虫类 fetcher（微博热搜、今日热榜、财联社）

每类接入方式写一个通用模板，新数据源只需配置即可接入。

**验收：** 每个 fetcher 独立运行，数据写入 RawContent 表，FetchLog 有完整日志。

---

### Phase 3：AI 处理层（1-2 天）

DeepSeek 职责：
1. **去重排名**：将一批原始新闻去重，按重要性评分排序
2. **摘要生成**：生成统一格式的 100 字中文摘要
3. **分类打标**：细分二级标签
4. **翻译**：英文内容翻译为中文
5. **周报生成**：一周精选新闻 → 结构化 Markdown 周报

处理流程：采集完成后触发 AI 处理 → 结果写入 ProcessedContent 表。

**验收：** 手动触发 AI 处理，ProcessedContent 表有正确的去重、排名、摘要数据。

---

### Phase 4：后端 API（1 天）

核心接口：
- `GET /api/news?category=domestic&page=1&limit=20` — 精选新闻列表
- `GET /api/news/raw?source=weibo` — 原始数据（调试用）
- `GET /api/github/trending?date=xxx`
- `GET /api/weekly?week=2025-W01`
- `POST /api/cron/trigger` — 手动触发采集+处理（需鉴权）
- `GET /api/sources` — 数据源状态概览

**验收：** 接口返回 AI 处理后的精选内容，分页正确，响应 < 500ms。

---

### Phase 5：前端页面（3-4 天）

1. Dashboard 首页 — 各板块 Top 3 摘要卡片 + 数据源状态
2. 新闻列表页 — 按板块/标签/日期筛选，显示 AI 评分和摘要
3. GitHub 热榜页 — 仓库卡片，按语言/日期筛选
4. 周报页 — Markdown 渲染，周度切换
5. 数据源管理页（简易版）— 查看各源运行状态和最近日志

**验收：** 响应式布局正常，暗色/亮色切换，空/加载/错误状态齐全。

---

### Phase 6：部署与监控（1 天）

- Docker 化或 Vercel 部署
- 定时任务：采集（每日 3 次）→ AI 处理（采集后触发）→ 周报（每周一）
- 简易监控：数据源健康状态面板

**验收：** 线上运行正常，定时任务按计划执行。

---

## 四、扩展规划

数据库设计已预留扩展能力，后续可添加：
- 新数据源：只需新增 fetcher + DataSource 表记录
- 新板块：只需新增 category 枚举值
- 用户偏好：添加 UserPreference 表，个性化推荐
- 多语言：ProcessedContent 已有 language 字段
- 评论/收藏：添加 UserAction 表
- 通知推送：基于 AI 评分阈值触发

## 五、关键原则

1. **原始数据短暂保留** — RawContent 存 Raw DB，14 天后自动清理，精选内容存 Main DB 长期保留
2. **AI 处理可重跑** — ProcessedContent 可以清空重新生成（只要 Raw DB 中原始数据未过期）
3. **来源可追溯** — ProcessedContent 冗余 sourceId + sourceName，原始数据清理后仍可显示来源
4. **数据源可插拔** — 新增/下线数据源不影响系统其他部分
5. **先跑通再扩展** — 每类接入方式先做一个，验证后再批量复制
6. **日志优先** — 采集和 AI 处理都必须有完整日志
